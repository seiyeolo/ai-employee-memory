# (주)퍼티스트 AI 조직 첫 부서 회의 안건

> 일시: 2026-03-29 (조직 구축 다음날)
> 참석: 세미에이전트(오케스트레이터) + 5개 부서장
> 목적: "설치 완료 → 실전 운영" 전환

---

## 왜 하는가?

AI 조직 6개 에이전트가 세팅됐다. 하지만 **데이터 없는 에이전트는 빈 껍데기**.
각 부서가 실제로 일하려면:
1. 파이오니어에게 **어떤 데이터**를 받아야 하는지 확정
2. 부서끼리 **어떤 데이터를 주고받는지** 확정
3. 4/1 새 창고 오픈에 **당장 뭘 해야 하는지** 확정

## 무엇을 논의하는가?

### 안건 1: 부서 간 데이터 흐름 확정 (온톨로지 기반)

```
[주문 들어옴]
  ↓
영업마케팅부: 주문 확인 → session_send →
  ↓
생산관리부: 재고 확인 → 있으면 → session_send →
  ↓
물류부: 포장 → 발송 → 송장번호 → session_send →
  ↓
영업마케팅부: 고객에게 발송 알림
  ↓
재무인사부: 매출 기록
```

**확정해야 할 것:**
- 각 화살표(→)에서 **누가 먼저 움직이는가?**
- 재고 부족 시 **어디서 분기하는가?**
- 긴급 주문 시 **절차 단축 가능한가?**

### 안건 2: 파이오니어에게 요청할 데이터 우선순위

| 순위 | 데이터 | 해당 부서 | 이유 |
|------|--------|----------|------|
| 1 | 제품별 가격표 | 영업 | 견적 못 냄 |
| 2 | 제품별 BOM | 생산 | 부품 소요량 모름 |
| 3 | 안전재고 기준 | 생산 | 경고 알림 못 함 |
| 4 | 택배 요금표 | 물류 | 배송비 계산 불가 |
| 5 | 직원 급여 정보 | 재무 | 급여 관리 불가 |
| 6 | 브랜딩 가이드 | 콘텐츠 | 통일된 톤 없음 |
| 7 | CS 매뉴얼 | 영업 | 고객 대응 기준 없음 |
| 8 | 매출/지출 데이터 | 재무 | 재무 분석 불가 |

### 안건 3: 4/1 새 창고 오픈 대비

```
D-3 (3/29 오늘)
├── 생산관리부: 현재 재고 실사 → 수량 확정
├── 물류부: 포장재/발송 자재 리스트업
└── 재무인사부: 자산 대장 초기화

D-2 (3/30)
├── 생산관리부: 이동 대상 재고 분류 (완제품 vs 부품)

D-1 (3/31)
├── 전 부서: 최종 체크리스트 확인
└── 오케스트레이터: 파이오니어에게 종합 보고

D-Day (4/1)
├── 생산관리부: 새 창고 재고 초기값 입력
├── 물류부: 새 창고에서 첫 발송 테스트
└── 재무인사부: 자산 이동 기록
```

### 안건 4: session_send 실제 적용 규칙

**부서 간 통신 프로토콜:**

| 보내는 부서 | 받는 부서 | 트리거 |
|------------|----------|--------|
| 영업 → 생산 | 재고 확인 요청 | 주문 접수 시 |
| 생산 → 물류 | 출고 지시 | 재고 확인 완료 시 |
| 물류 → 영업 | 송장번호 전달 | 발송 완료 시 |
| 물류 → 재무 | 출고 건수 보고 | 일일 마감 시 |
| 영업 → 재무 | 매출 발생 보고 | 주문 확정 시 |
| 생산 → 재무 | 입고 비용 보고 | 부품 입고 시 |
| 콘텐츠 → 영업 | 프로모션 일정 공유 | 캠페인 기획 시 |
| 영업 → 콘텐츠 | 고객 후기/데이터 공유 | 월 1회 |

**모든 session_send는 오케스트레이터(세미에이전트)를 경유한다.**
