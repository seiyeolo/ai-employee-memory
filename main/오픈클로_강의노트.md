## 영상 1: 90%가 모르는 기능 (크롤링, 하네스)
**URL:** https://youtu.be/wNdgL2S5078

### 핵심 내용

**1. 릴레이 대체 → Chrome DevTools MCP**
- 릴레이 기능이 사라짐 → Chrome DevTools MCP로 대체
- `chrome://inspect` → Remote Debugging 켜기
- openclaw.json의 MCP 서버에 추가:
  - `npx chrome-devs-mcp` + `--auto-connect` 옵션 필수
- 게이트웨이 재시작 후 사용

**2. 웹 크롤링 실습 (경매 사이트)**
- 텔레그램에서 "대전 지역 5억 이하 경매 매물 아파트 뽑아줘" 지시
- OpenClaw가 브라우저 자동 제어 → 조건 입력 → 데이터 추출 → 엑셀 생성
- 세션 타임아웃 길이를 늘려야 안정적
- 크론잡으로 정기 자동화 가능

**3. 스킬 구조**
- SKILL.md 하나 = 자동화된 업무 하나
- ClawHub에서 스킬 설치 가능 (보안 주의 — 5~25% 보안 우려 스킬 존재)

**4. 업데이트 현황 (2026.3.28 기준)**
- Teams STK 전환 (Slack처럼 Teams 지원)
- ClawHub 기반 스킬 설치 방향
- ACP 채널 바인딩 (Codex 워크스페이스 연결)
- 세션 초기화: `/new` 명령

**5. 실전 팁**
- `verbose` 켜면 툴콜링 과정 실시간 확인 가능
- `stop`으로 작업 중단
- 한 번에 100~200건 크롤링이 안정적

## 영상 2: 비용 90% 줄이는 법
**URL:** https://www.youtube.com/live/qSriDgNqy0A

### 핵심 내용

**1. OpenRouter 무료 토큰 사용법**
- openrouter.ai 가입 → API 키 발급
- 무료 모델 존재 (Free 표시): Nemotron 3 Super 등
- 무료 = 데이터 피딩 (개인식별정보 PII 넣지 말 것)
- Google AI Studio도 동일 원리

**2. OpenClaw에 OpenRouter 연결**
- `openclaw config` → 모델 프로바이더 → OpenRouter 추가
- 여러 프로바이더를 동시에 설정 가능 (무료 모델 여러개 등록)
- `onboard`는 전체 변경, `config`는 옵션 추가

**3. 토큰 아껴쓰는 법**
- **하트비트 끄기** — 메인 빼고 전부 OFF → 토큰 대폭 절감
- 에이전트마다 하트비트 돌리면 토큰 많이 소모
- 기본 모델: GLM 5 (저비용)

**4. 핵심 명령어**
- `openclaw status` — 가장 중요한 명령. 모든 정보 확인
- `openclaw config` — 설정 변경
- `openclaw onboard` — 전체 기본 설정 변경

**5. 보안 주의**
- ClawHub 스킬 설치 전 항상 검사
- 공유기 환경에서는 외부망 차단으로 비교적 안전
- 이상한 스킬 설치가 가장 큰 보안 위험

## 뽀짝이 사례 (지피터스 AI스터디)
**출처:** ~/Downloads/뽀짝.pdf (송다혜님 사례, 4페이지)

### 배경
- 지피터스 AI스터디: 매 기수 450명, 18개 스터디, 운영자 2명
- "이번에도 이걸 우리 둘이서 다 해야 해?" → AI 에이전트 "뽀짝이" 도입

### 핵심 교훈 5가지

**1. 페르소나 설계 = 사용 허들 낮추기**
- "AI 시스템에 작업을 요청합니다" → "뽀짝아, 이거 해줘"
- 봄베이 종 까만 고양이 캐릭터 → 팀원들이 부담 없이 사용
- 업무 매뉴얼(SKILL.md) 하나 = 자동화된 업무 하나

**2. 감정적 신뢰 구축**
- 핵심은 "해줄 수 있는 것의 범위가 달라진 것"
- 예: 감사 쿠폰 발급 — 기존 여러 단계 → Slack에서 한마디로 끝
- CS가 고객 문의에서 동료 의견으로 바뀜

**3. 팀 협업 시뮬레이션: "뽀짝이한테 물어봐"**
- 새 팀원 온보딩 — 담당자가 아닌 에이전트에게 직접 질문
- 조직의 지식이 사람 머릿속이 아닌 에이전트에 축적

**4. "해보자!" 마인드**
- 기존: "이걸 어떻게 구현하지?" → 포기
- 도입 후: "해줄 수 있다는 확신" → 시도 증가
- OT 자기소개 영상, 파트너 대시보드 등 즉시 제작

**5. 에이전트 네이티브 운영**
- 기존 도구에 자동화를 얹는 것 ❌
- 처음부터 에이전트가 할 수 있는 방식으로 설계 ✅
- Google Slides API로 스터디별 자동 반영 구조

### 실수와 성장
- 404 링크 문자 255명 발송
- https:// 누락 링크 287명 발송
- 댓글 24개 중복 → 일괄 삭제
- **사고마다 규칙 1개 추가 → 15개 이상 축적**
- "같은 실수를 두 번 하지 않게 만드는 시스템"

## 마인드맵 구조: 오픈클로 업데이트 및 주요 기능

```
오픈클로 업데이트 및 주요 기능
├── 주요 업데이트 사항
│   ├── 3월 28일 업데이트
│   ├── 팀즈 SDK 전환 및 전면 지원
│   ├── 클로허브 중심 스킬 설치
│   ├── UI 변경 및 노드 버전 업데이트
│   └── 보안 강화 및 버그 수정
├── 신규 및 강화 기능
│   ├── ACP 채널 바인딩 (디스코드, 아이메시지)
│   ├── 그록 웹 기능 및 플러그인 도구 승인
│   ├── 슬랙 파일 업로드 및 매트릭스 TTS
│   └── 오픈클로 제이슨 스키마 확인
├── 크롬 데브툴 MCP (릴레이 대체)
│   ├── 설치: 크롬 인스펙트 리모트 디버깅 + auto-connect
│   └── 활용: 유튜브 크롤링, 경매 검색, 엑셀 변환
├── 오픈클로 하네스 (Harness)
│   ├── 개념: 목표 도달까지 에이전트 반복 훈련
│   ├── 구조: 계획 → 실행 → 리뷰 오케스트레이션
│   ├── 서브 에이전트 및 스폰 기능 활용
│   └── 효율적 워크플로우 구현
├── 모델 및 API 활용 전략
│   ├── Z.ai (GLM) API: 저렴하고 고성능
│   ├── ChatGPT 오어스 로그인 방식
│   ├── 모델 라우팅 및 폴백 구성
│   └── 예산 설정 및 장시간 에이전트 운용
└── 실제 연구 및 자동화 사례
    ├── 오토 리서치 크루 기반 논문 작성
    ├── 위스퍼 한국어 음성인식 연구
    └── AI 상호 학습 및 튜닝 자동화
```

### 핵심 주장 5가지 + 숨은 가정

| # | 주요 주장 | 근거 | 숨은 가정 / 오해 |
|---|---------|------|---------------|
| 1 | 릴레이 대신 크롬 데브툴 MCP | 브라우저 직접 제어 가능 | **auto-connect 설정 필수** (설치만으로 안 됨) |
| 2 | 하네스는 능동적 워크플로우 | 목표까지 반복 훈련 | **명확한 Target 필수** (없으면 작동 안 함) |
| 3 | 서브에이전트 병렬 처리 | 스폰으로 알바생 에이전트 | **비용 최적화 전략** (메인 부담 줄이기) |
| 4 | 저렴한 모델(GLM)로 고성능 | Claude 급 성능, 비용 저렴 | **작업별 모델 라우팅 필요** (전부 고가 모델 ❌) |
| 5 | 복잡한 웹은 세션 타임 조절 | 기본 타임아웃이 짧음 | **늘려야 안정적** (경매 사이트 등) |

## OpenClaw School 101 (4차시 강의)
**출처:** https://github.com/OpenClawSchool/OpenClaw-101
**로컬:** `~/clawd/knowledge/OpenClaw-101/`

### 1차시: 나의 AI 비서와 인사하기

| 항목 | 내용 |
|------|------|
| 핵심 | 오픈클로 설치 → 메신저 연동 → 첫 대화 |
| AI 발전 흐름 | GPT(질답) → Copilot(보조) → Agent(자율실행) |
| OpenClaw 탄생 | Clawdbot → Moltbot → OpenClaw (2026.01) |
| 창립자 | Peter Steinberger → OpenAI 합류 (2026.02) |
| 라이선스 | MIT (무료, 상업 사용 가능) |
| 준비물 | GitHub 계정, GCP/로컬 환경, Maton API, 텔레그램 |

### 2차시: 전문가 Knowledge Base 구축

| 항목 | 내용 |
|------|------|
| 핵심 | 모델 성능보다 "어떤 데이터를 주느냐"가 중요 |
| 핵심 파일 8개 | USER, SOUL, IDENTITY, AGENTS, MEMORY, HEARTBEAT, BOOTSTRAP, TOOLS |
| KB 구축 | Obsidian + Markdown + RAG |
| AI 친화적 구조 | Topic → Concept → Example → Checklist |
| 세션 개념 | 종료 시 MEMORY.md로 요약 저장. 파일 기반 기억은 영구 유지 |

### 3차시: 개인비서에서 직원으로 전환

| 항목 | 내용 |
|------|------|
| 핵심 | 비서(Reactive) → 직원(Proactive) 격상 |
| 차이 | 비서=정보 제공, 직원=SOP 기반 자율 실행 |
| GitHub Second Brain | 지식+SOP 보관 → 어디서든 동기화 |
| 근무일지 | 크론잡으로 매일 자동 작성 → 업무 가시성 확보 |
| 서브에이전트 | 전문 업무 위임 및 협업 설계 |
| Fine-grained PAT | Contents: Read and Write 권한 토큰 |

### 4차시: 마켓 등록하기 (상품화)

| 항목 | 내용 |
|------|------|
| 핵심 | 에이전트 패키징 → 브랜딩 → 마켓플레이스 등록 |
| 알바봇 플랫폼 | 크몽/숨고(사람 시간) ↔ 알바봇(AI 에이전트가 일함) |
| 등록 방식 3가지 | API, MCP 서버, 오픈클로 채팅 |
| AI 윤리 7원칙 | 생명보호, 정서적안전, 관계우선, 정직성, 샌드박스, 인증보관, 이상징후탐지 |
| 보안 수칙 | API키 보호, 보안 샌드박스, 최소 권한 원칙 |
| 수익 구조 | 에이전트 이코노미 — 훈련된 에이전트가 수익 창출 |

### 퍼티스트와 강의 커리큘럼 대응
