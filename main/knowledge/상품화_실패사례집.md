## 사례 1: 서브에이전트를 만들었는데 오케스트레이션이 안 된다

**상황:** 6개 부서 에이전트를 만들고, AGENTS.md에 "session_send로 위임한다"고 써놨다.
**기대:** 텔레그램에서 "재고 보고해줘" → 생산관리부에 자동 위임
**현실:** "세션 가시성 제한으로 부서에 메시지를 보낼 수 없습니다"

**원인:** openclaw.json에서 main의 subagents로 등록하지 않았다. 에이전트를 만든 것과 연결한 것은 별도 작업.
**교훈:** **만들기 ≠ 연결하기.** 설치 스크립트에 서브에이전트 자동 연결을 포함해야 한다.
**코난쌤 표 대응:** #3 서브에이전트 스폰 — 메인의 부담 줄이기

## 사례 2: AGENTS.md에 규칙을 써놨는데 에이전트가 안 따른다

**상황:** "재고/생산 관련 → production에 위임"이라고 써놨다.
**기대:** 에이전트가 규칙을 읽고 자동으로 위임
**현실:** 직접 파일을 찾으려다 실패. 위임 안 함.

**원인 2가지:**
1. 도구 이름을 `session_send`로 썼는데 실제는 `sessions_send`
2. 재고 파일 경로(`~/clawd-production/data/재고현황.md`)를 명시하지 않음

**교훈:** **개념(온톨로지) ≠ 구현(도구 호출).** 설계 문서에 실제 도구명과 파라미터를 반드시 명시. 파일 경로는 절대경로로.
**코난쌤 표 대응:** #2 하네스 — 명확한 Target 설정 필수

## 사례 3: 설정을 바꿨는데 적용이 안 된다

**상황:** openclaw.json에 서브에이전트 추가, AGENTS.md에 파일 경로 추가 완료.
**기대:** 바로 적용
**현실:** 여전히 "파일을 찾을 수 없습니다"

**원인 2가지:**
1. 게이트웨이 재시작 안 함 → `openclaw gateway restart` 필요
2. 기존 세션이 살아있어서 새 AGENTS.md를 안 읽음 → 세션 리셋 필요

**교훈:** **설정 변경 = 반드시 재시작 + 세션 리셋.** 이걸 빠뜨리면 "왜 안 되지?"로 시간만 낭비.
**코난쌤 표 대응:** #5 세션 타임 조절 관건 — 세션 관리의 중요성

## 사례 4: GitHub repo를 잘못 만들어서 기존 시스템이 끊겼다

**상황:** AI 조직 파일을 GitHub에 올리려고 새 repo(puttist-ai-org)를 만들었다.
**기대:** 깔끔한 관리
**현실:** 기존 ai-employee-memory에 이미 같은 구조가 있었고, 텔레그램 KB 알림이 "변경 없음"만 계속 나옴.

**원인:** 기존 인프라(GitHub repo, 텔레그램 알림, 크론잡) 연결 상태를 확인하지 않았다.
**교훈:** **새로 만들기 전에 기존 연결부터 확인.** 노드를 옮기기 전에 엣지(연결)부터 파악.
**코난쌤 표 대응:** #1 auto-connect — 설치만으로 끝이 아님

## 사례 5: Claude Code 호출이 command not found

**상황:** AGENTS.md에 `exec: claude --print "..."` 호출 규칙을 써놨다.
**기대:** OpenClaw가 Claude Code를 자동 호출
**현실:** PATH에 claude가 없어서 실행 실패

**원인:** npm global 설치 경로(`~/.npm-global/bin/`)가 OpenClaw의 exec 환경 PATH에 포함 안 됨.
**해결:** 전체 경로 `/Users/mac/.npm-global/bin/claude --print "..."` 로 변경
**교훈:** **exec 호출 시 항상 절대경로.** 설치 환경마다 PATH가 다르므로 자동 감지 필요.
**코난쌤 표 대응:** #1 auto-connect 설정 필수 — 숨은 설정의 중요성
