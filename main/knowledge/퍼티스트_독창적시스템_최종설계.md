# (주)퍼티스트 독창적 AI 운영 시스템 — 최종 설계

> 원툴 완전 벤치마킹 + 온톨로지 로직 + 랭그래프적 사고
> "원툴이 뼈대를 팔았다면, 우리는 뼈대에 AI 근육과 신경을 붙인다"

---

## 온톨로지 4축 (원툴 벤치마킹 → 퍼티스트 확장)

### 원툴의 4축
```
학습/결과관리 | 업무관리 | 목표관리 | 조직관리
```

### 퍼티스트의 6축 (4축 + 2축 추가)
```
1. 업무관리     — 누가 무엇을 언제까지 (업무 DB)
2. 생산/재고    — 만들고 세고 보관하기 (BOM, 재고, 창고)
3. 상거래       — 사고 팔기 (주문, 출고, 매출, AS)
4. 목표/성과    — 어디로 가고 있는가 (OKR, KPI)
5. 조직/인사    — 누가 일하는가 (사람 + AI 에이전트)
6. 지식/학습    — 무엇을 알고 있는가 (KB, 기억, 피드백)
```

## 온톨로지 그래프 (엔티티 + 관계)

```
                    [목표 OKR]
                   ↗         ↘
            [KPI 지표]      [사업 계획]
               ↑                ↑
          [매출 데이터]     [시장 조사]
               ↑                ↑
    ┌─────[주문]─────────[고객/채널]
    │          ↑                ↑
    │     [출고/납품]      [콘텐츠]──→[9개 채널]
    │          ↑                ↑
    │     [완제품 재고]    [Knowledge Base]
    │          ↑                ↑
    │     [생산]──[BOM]    [학습/피드백]
    │          ↑                ↑
    │     [부품 재고]      [기억 (GitHub)]
    │          ↑
    │     [입고/발주]
    │          ↑
    └─────[거래처]
    
    가로축: 상거래 흐름 (왼쪽→오른쪽)
    세로축: 생산 흐름 (아래→위)
    대각축: 지식 흐름 (KB→콘텐츠→채널)
```

## 에이전트 × 온톨로지 매핑

각 에이전트가 온톨로지의 어떤 노드를 소유/관리하는지:

```
세미에이전트 (오케스트레이터)
  소유: [목표OKR], [KPI지표], [사업계획], [업무배분]
  접근: 모든 노드 (읽기)
  권한: 부서간 데이터 요청, 최종 승인

생산관리부장
  소유: [BOM], [부품재고], [완제품재고], [생산], [입고], [창고]
  접근: [거래처](읽기), [출고](재고차감 수신)
  트리거: 안전재고 이하 → 발주 추천

물류부장
  소유: [출고], [납품], [택배], [배송추적]
  접근: [완제품재고](확인), [주문](수신)
  트리거: 출고 완료 → 매출 데이터 전달

영업마케팅부장
  소유: [주문], [고객/채널], [AS], [거래처], [견적]
  접근: [완제품재고](확인), [매출](생성)
  트리거: 신규주문 → 재고확인 → 출고지시

콘텐츠부장
  소유: [콘텐츠], [9개채널], [트렌드분석]
  접근: [KB](참조), [고객후기](참조)
  트리거: 주간 기획안 → 승인 → 제작 → 발행

재무인사부장
  소유: [매출장부], [지출장부], [손익], [급여], [세무]
  접근: [매출](수신), [매입](수신), [인사](관리)
  트리거: 월말 → 손익 자동 생성
```

## 랭그래프 흐름 (조건 분기 + 순환)

### 핵심 흐름 1: 생산→판매 사이클
```
[거래처] →(입고)→ [부품재고] →(생산)→ [완제품재고]
                                          ↓
[고객] ←(배송)← [출고] ←(주문확인)← [주문접수]
                  ↓
              [매출기록] → [손익계산] → [KPI] → [OKR 달성도]
```

### 핵심 흐름 2: 피드백 순환
```
[고객] →(AS접수)→ [AS처리] →(패턴분석)→ [품질개선]
                                           ↓
                               [생산공정 수정] → [제품 개선]
                                           ↓
                               [콘텐츠 반영] → [고객 소통]
```

### 핵심 흐름 3: 지식 순환
```
[일상 업무] →(기록)→ [memory/] →(정리)→ [MEMORY.md]
                                            ↓
                                    [Knowledge Base]
                                            ↓
                                    [에이전트 스킬 향상]
                                            ↓
                                    [더 나은 업무 처리]
                                            ↓
                                    [일상 업무] (순환)
```

## 독창성: 원툴과의 결정적 차이

| 관점 | 원툴 (노션) | 퍼티스트 (OpenClaw) |
|------|-----------|-------------------|
| **입력** | 사람이 타이핑 | 텔레그램에 말하기 |
| **처리** | 사람이 분류/정리 | AI가 온톨로지 따라 자동 |
| **연동** | 수동 관계 설정 | BOM/재고/매출 자동 연쇄 |
| **알림** | 노션 알림 | 텔레그램 즉시 보고 |
| **학습** | 없음 (정적) | GitHub 기억 축적 (동적) |
| **확장** | 팀 추가 (수동) | 에이전트 추가 (자동) |
| **비용** | 노션 팀플랜 (인당 $10) | 로컬 모델 무료 + API 소액 |
| **글로벌** | 다국어 수동 | AI 자동 번역/현지화 |
