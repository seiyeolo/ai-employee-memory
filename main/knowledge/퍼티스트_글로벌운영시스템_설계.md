# (주)퍼티스트 글로벌 운영 시스템 설계

> 원툴(워킨온) 벤치마킹 + OpenClaw AI 에이전트 기반
> 미국/일본/중국 진출 대비 확장 가능 구조

---

## 벤치마킹: 원툴 vs 퍼티스트

| 원툴 (노션 기반) | 퍼티스트 (OpenClaw 기반) | 차이점 |
|---------------|---------------------|-------|
| 사람이 데이터 입력 | 텔레그램에 말하면 AI가 기록 | 입력 자동화 |
| 노션 DB 중심 | 마크다운/CSV + GitHub | 도구 독립적 |
| 웹에서 확인 | 텔레그램으로 보고 받음 | 이동 중 확인 |
| 팀원이 직접 업데이트 | AI 에이전트가 업데이트 | 인력 불필요 |
| 월 비용 (노션 팀) | 로컬 모델 무료 + API 소액 | 비용 절감 |

## 시스템 구조 (6개 부서 + 글로벌 확장)

```
파이오니어 (총괄이사)
    │ 텔레그램 (한국어)
    │
세미에이전트 (오케스트레이터)
    │
    ├── 생산관리부 ──── 생산/재고/BOM/품질/새 창고
    ├── 물류부 ──────── 출고/택배/납품/해외배송
    ├── 영업마케팅부 ── CRM/견적/AS/시장조사
    ├── 콘텐츠부 ────── 9개 채널/다국어 콘텐츠
    ├── 재무인사부 ──── 매출/지출/급여/세무/예산
    │
    └── [확장] 글로벌사업부 (향후)
        ├── 미국팀 ── Amazon US/영어 CS
        ├── 일본팀 ── 일본 대리점/일어 CS
        └── 중국팀 ── 중국 시장 진출

```

## 원툴 벤치마킹 → 퍼티스트 적용

### 1. 업무 DB → 업무 추적 시스템

원툴의 업무 DB 컬럼을 퍼티스트에 맞게 변환:

| 원툴 컬럼 | 퍼티스트 적용 | 담당 에이전트 |
|---------|-----------|-----------|
| :::팀 | 부서 (6개) | 오케스트레이터 |
| :::업무 레벨 | 업무 유형 (일상/프로젝트/긴급) | 각 부서장 |
| ::우선순위 긴급도 | 긴급/일반/낮음 | 오케스트레이터 |
| :::제목 | 업무 제목 | 각 부서장 |
| :::상태 | 시작전/진행중/완료/보류 | 각 부서장 |
| :::담당자 | AI 에이전트 + 사람 | 오케스트레이터 |
| :::시작일/종료일 | 기한 관리 | 크론 알림 |
| :연관 회의록 | 대화 기록 (memory/) | GitHub |
| :연관 피드백 | 텔레그램 보고 이력 | memory/ |
| :OKR 연동 | KPI 대시보드 | 오케스트레이터 |

### 2. 영업팀 보드 → 영업마케팅부 스킬

```
영업팀 위키 → 영업마케팅부 knowledge/
├── 매출 보고서 → 월간 매출 자동 생성 (크론)
├── 영업 실적 보고서 → 스마트스토어+Amazon 자동 집계
├── 영업 계획서 → 분기별 영업 계획 (수동)
└── 시장 조사 보고서 → ClawFlows competitor-shadow + SEO
```

### 3. 매출관리 대시보드 → 재무인사부 보고

```
매출/매입 현황 → 재무인사부 월간 손익 자동 생성
세금 계산서 → 세무 일정 크론 알림
거래처 정보 → 거래처마스터.csv
거래 내역 → 매출장부/지출장부 CSV
```

### 4. CEO보드 → 오케스트레이터 주간 종합 보고

```
목표 & 매출 현황 → KPI 대시보드 (주간)
목표 OBJ DB → OKR 관리 (분기별)
월별 매출 비중 → 제품별/채널별 매출 분석
팀별 보드 → 6개 부서 현황 종합
```

## 글로벌 확장 설계 (미국/일본/중국)

### Phase 1: 현재 (한국 중심)
```
판매: 스마트스토어 (국내) + Amazon US (해외)
CS: 한국어 A/S 폼 (Google Apps Script)
콘텐츠: 한국어 9개 채널
```

### Phase 2: 미국 강화 (다음 단계)
```
판매: Amazon US 최적화, 자체 웹사이트 영문화
CS: 영어 A/S 대응 (AI 자동 번역 + 대응)
콘텐츠: 영어 유튜브, Instagram 영문 콘텐츠
물류: 미국 FBA 또는 3PL 활용
에이전트: 콘텐츠부에 영문 스킬 추가
```

### Phase 3: 일본 진출
```
판매: 일본 Amazon + 대리점
CS: 일본어 A/S (AI 번역)
콘텐츠: 일본어 콘텐츠 (퍼티스트 파크 = 파크골프 강국)
물류: 일본향 제품 전용 라인 (이미 존재)
에이전트: 일본시장 전담 스킬
```

### Phase 4: 중국 진출
```
판매: 중국 플랫폼 (타오바오, JD)
CS: 중국어 대응
콘텐츠: 위챗, 도우인
에이전트: 중국시장 전담 스킬
```

## 상품화 가능성 (SaaS)

이 시스템 자체를 **다른 중소기업에 판매** 가능:

```
상품명: "AI 직원 조직 구축 키트" 또는 "OpenClaw 비즈니스 팩"

포함 내용:
├── 6개 부서 에이전트 설정 템플릿
├── 업무 추적 시스템 (원툴 구조 기반)
├── 재무/인사 관리 양식
├── Knowledge Base 구축 가이드
├── 텔레그램 연동 설정 가이드
├── GitHub 기억 저장소 설정
└── 비용 최적화 가이드 (로컬 모델)

판매 채널:
├── 와디즈 (크라우드펀딩)
├── 자체 웹사이트
├── OpenClaw School 연계
└── 강의/컨설팅

가격대:
├── 셀프 설치 키트: 10~30만원
├── 설치 대행 + 교육: 50~100만원
└── 월 구독 (관리형): 월 10~30만원
```

## 핵심 정리

원툴(노션)이 "500억 회사의 시스템"을 템플릿으로 팔았다면,
퍼티스트는 "**AI가 알아서 일하는 회사 시스템**"을 실전 경험과 함께 판다.

차이점:
- 원툴: 사람이 입력 → 노션이 정리
- 퍼티스트: 텔레그램에 말하기 → AI가 입력+정리+보고+실행

이것이 **"비서에서 직원으로"**의 실제 구현체이며,
동시에 **상품**이기도 하다.
