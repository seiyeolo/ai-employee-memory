# 프로젝트 중 주요 의사결정 기록

## 결정 1: 조직 구조
- **결정:** 4개 부서(생산관리/물류/영업마케팅/콘텐츠) + 오케스트레이터
- **이유:** 5인 소기업에서 AI가 부서별 전문성을 가지고 반복 업무 자동화
- **대안:** 에이전트 1개로 모든 업무 처리 → 맥락 혼란, 비효율적

## 결정 2: 데이터 관리 방식 (B방식)
- **결정:** OpenClaw 워크스페이스 중심 운영 (노션 아님)
- **이유:** A/S는 Google Apps Script, 나머지는 마크다운/CSV → 노션 불필요
- **대안:** A방식(노션 허브) → 노션 학습 어려움, 이미 다른 시스템 작동 중

## 결정 3: 모델 비용 최적화
- **1차:** 3개 부서 → ollama/qwen3-vl:32b (무료, 로컬)
- **2차:** 오케스트레이터+콘텐츠 → openai/gpt-4o ($100 크레딧)
- **이유:** Anthropic 크레딧 소진, OpenAI $100 크레딧 보유
- **결과:** 일일 크론 비용 약 90% 절감

## 결정 4: GitHub 기억 저장소
- **결정:** seiyeolo/ai-employee-memory (Private)
- **이유:** 3대 장비 기억 동기화 + 백업 필수
- **파이오니어 인사이트:** "GitHub가 AI 직원의 필수 인프라"

## 결정 5: 콘텐츠부 독립
- **결정:** 영업마케팅부에서 분리하여 독립 부서로 생성
- **이유:** 퍼티스트 골프 + 퍼티스트 파크 2개 제품라인, 9개 채널 관리
- **파이오니어 발언:** "콘텐츠팀은 별도로 조직하는게 좋을 것 같습니다"

## 결정 6: 인사관리 직접 관리
- **결정:** 별도 인사부 없이 오케스트레이터가 직접 관리
- **이유:** 5인 소기업, 인사 업무 빈도 낮음
- **구현:** 크론으로 정기 알림 (임금명세서, 연차 점검)

## 결정 7: 경쟁사 가격 조사 = 정기 업무 아님
- **결정:** 네이버 쇼핑 API 설치하되 크론으로 자동화하지 않음
- **이유:** 필요할 때만 시켜서 쓰는 도구
- **파이오니어 발언:** "정기업무는 아닙니다"

## 파이오니어의 사고 방식 (y=f(x))
- y(목표/상수): 달성해야 할 것 — 고정
- x(변수/입력): 변할 수 있는 것 — 인정
- f(함수/전략): 대응 방법 — 조정
→ "목표는 지키고, 변수에 따라 방법을 바꾼다"
