# AI 비서에서 직원으로의 전환 — 핵심 개념

## 비서 vs 직원의 차이

| 구분 | 비서 AI | 직원 AI |
|------|---------|---------|
| 반응 방식 | 질문에만 반응 (수동) | 목표를 향해 주도적 행동 |
| 기억 | 대화 끝나면 까먹음 | 지식베이스에 경험 축적, 성장 |
| 도구 | 제한적 | 복잡한 도구 능숙하게 사용 |
| 업무 | 혼자 모든 일 | 팀워크, 위임 |
| 승인 | 승인 없이 실행 | 중요 결정 전 승인 요청 |

## 직원 AI 만들기 3단계

### 1단계: 장기 기억 (Second Brain)
- GitHub 비공개 저장소 생성
- AI 전용 PAT(접근 토큰) 발급
- 작업 공간과 저장소 연결
→ "영구적인 기억 장치"

### 2단계: 업무 규칙 (SOP)
- ClawFlows / Lobster Engine = 표준 업무 절차
- 제로 토큰 비용 (정해진 절차 따름)
- 결정론적 실행 (매번 같은 결과)
- 승인 게이트 (위험한 작업 전 사용자 확인)

### 3단계: 위임 (서브에이전트)
- 메인 AI = 팀장
- 전문 분야별 서브에이전트 = 팀원
- 채용 공고 템플릿으로 서브에이전트 생성

## 5가지 핵심 원칙 (인사 평가)
1. **중단 복원** — 작업 중단 시 이어서 할 수 있는가?
2. **업무 기록** — 모든 과정을 근무일지처럼 기록하는가?
3. **도구 선택** — 상황에 맞는 최적의 도구를 꺼내쓰는가?
4. **승인 요청** — 중요 결정 전 상사에게 승인 요청하는가?
5. **지식 축적** — 해결한 문제를 기록해 다음에 더 잘하는가?

## 핵심 문장
> "에이전트가 직원이 된다는 것은 내가 일일이 지시하지 않아도
> 정의된 프로세스에 따라 스스로 결과를 만들어 낸다는 것을 의미한다."

## Knowledge Base의 중요성
- 모델 성능(Gemini/GPT)보다 중요한 것은 '어떤 데이터'를 주느냐
- LLM 일반 상식 + 비공개 전문 지식 = 진짜 전문가
- RAG: 수많은 문서 중 필요한 것만 골라서 읽는 방식
