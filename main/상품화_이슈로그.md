## #001 | 2026-03-29 | 서브에이전트 미연결

**문제:** 부서별 에이전트를 만들었지만 openclaw.json에서 main의 서브에이전트로 등록하지 않음. 오케스트레이션(session_send) 불가.
**원인:** 에이전트 생성과 서브에이전트 연결이 별도 작업인데, 연결 단계를 빠뜨림.
**해결:** openclaw.json의 main 에이전트에 subagents.list 추가.
**일반화:** ✅ 모든 고객이 겪을 문제. 설치 스크립트에 자동 연결 포함 필요.

## #002 | 2026-03-29 | Claude Code PATH 문제

**문제:** AGENTS.md에 `claude --print`로 호출 지시했지만, OpenClaw exec 환경에서 PATH 미등록으로 command not found.
**원인:** npm global 설치 경로가 쉘 PATH에 포함 안 됨.

**일반화:** ✅ 설치 환경마다 경로 다름. 설치 시 자동 감지 필요.

## #005 | 2026-03-29 | sessions_send 도구 사용법 미명시

**일반화:** ✅ 설계와 구현 사이 연결 문서 필수. 판매 패키지에 포함.

## #006 | 2026-03-29 | 게이트웨이 재시작 안 하면 설정 미반영

**문제:** openclaw.json에 서브에이전트 추가했지만 게이트웨이 재시작 없이 적용 안 됨.

## #007 | 2026-03-29 | 세션 리셋 필요 (AGENTS.md 변경 후)

**문제:** AGENTS.md를 업데이트해도, 기존 세션이 살아있으면 새 내용을 안 읽음.
**원인:** OpenClaw는 세션 시작 시 AGENTS.md를 로드. 이미 시작된 세션은 이전 버전 유지.
**해결:** 세션 파일 삭제 + 게이트웨이 재시작 → 새 세션에서 최신 AGENTS.md 로드.

**성공 확인:** 텔레그램 "재고 보고해줘" → 재고현황.md 직접 읽어서 정상 보고 ✅

## #008 | 2026-03-30 | 크롬 리모트 디버깅 허용 시 블로그로 리다이렉트

**문제:** `chrome://inspect/#remote-debugging`에서 "허용" 클릭 → `developer.chrome.com/blog/chrome-devtools-mcp-debug-your-browser-session` 페이지로 자동 이동
**원인:** Chrome M144+의 정상 동작. 원격 디버깅 활성화 후 공식 MCP 설정 안내 페이지를 자동으로 열어줌.
**해결:** 오류 아님. 허용을 누른 시점에 이미 원격 디버깅은 활성화된 상태. 블로그는 "다음 단계 안내"일 뿐.
**일반화:** ✅ 고객이 "허용 눌렀는데 이상한 페이지가 떴다"고 문의할 수 있음. FAQ에 포함.
**참고 URL:** https://developer.chrome.com/blog/chrome-devtools-mcp-debug-your-browser-session?hl=ko

## #009 | 2026-03-30 | 크롬 데브툴 MCP "탭 없음" — 크롬 재시작 필요

**문제:** 텔레그램 "크롬 탭 보여줘" → "열려 있는 크롬 탭이 없습니다"
**원인:** `chrome://inspect`에서 "허용"을 눌렀지만, 크롬을 재시작하지 않으면 디버깅 포트가 실제로 열리지 않음.
**해결:** 크롬 완전 종료(Cmd+Q) → 다시 열기 → 디버깅 포트 활성화됨.
**일반화:** ✅ "허용 눌렀는데 안 돼요" 고객 문의 시 → "크롬 재시작하셨나요?" FAQ 필수.

## #010 | 2026-03-30 | 크롬 데브툴 MCP 설정됐지만 도구 미노출 (미해결)

**문제:** `openclaw mcp list`에 chrome-devtools 표시, 대시보드 Infrastructure > Mcp 탭에도 설정 확인됨. 그러나 Agents > Tools의 "Connected tools"에 chrome-devtools 도구가 안 나타남. 에이전트가 내장 browser 도구만 사용.
**시도한 것:**
1. openclaw.json에 `mcp.servers` 키로 수동 추가 → 실패
2. `openclaw mcp set` 명령으로 정식 등록 → 설정은 됐지만 도구 미노출
3. 2026.3.24 → 2026.3.28 업데이트 → 동일
4. 대시보드에서 Update(게이트웨이 재시작) → 동일
5. 크롬 리모트 디버깅 활성화 (127.0.0.1:9222) → 포트는 열렸지만 MCP 연결 안 됨
**원인 추정:** OpenClaw 게이트웨이가 MCP stdio 서버를 에이전트 도구로 연결하는 메커니즘 이해 부족. 공식 문서 또는 커뮤니티 확인 필요.
**상태:** 미해결 — 다음 단계: 코난쌤 강의 재확인, OpenClaw 공식 문서/커뮤니티 질문
**일반화:** ✅ MCP 설치가 가장 큰 진입 장벽. 판매 시 "MCP 설치 대행" 서비스 또는 원클릭 설치 스크립트 필수.

## #011 | 2026-03-30 | OpenClaw 업데이트 시 실행파일 삭제됨

**문제:** `npm update`로 업데이트 후 `openclaw` 실행파일이 사라짐 (`no such file or directory`).
**원인:** npm global 업데이트 과정에서 기존 바이너리가 삭제되고 새 버전이 설치되기 전 상태.
**해결:** `npm install -g openclaw@latest`로 재설치 → 2026.3.28 정상 설치.
**일반화:** ✅ 수강생/고객에게 반드시 안내해야 할 사항. 업데이트 가이드에 "실행파일 삭제 가능성 + 재설치 방법" 포함.
**주의:** 업데이트 중 게이트웨이가 중지되므로, 텔레그램 봇도 일시 중단됨. 업무 시간 외 업데이트 권장.
