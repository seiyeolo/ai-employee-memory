## #015 | 2026-03-30 | OpenClaw exec→Claude Code 호출 시 새 브라우저로 열림

1. OpenClaw의 exec 환경에서 Chrome DevTools MCP가 `--autoConnect` 대신 새 인스턴스를 시작하는 것일 수 있음
2. OpenClaw 자체의 내장 browser 도구가 먼저 실행되어 간섭하는 것일 수 있음

**해결 (2026-03-30):**

- 다만 작업 자체는 미완료 — 별도 원인 파악 필요
**일반화:** ✅ `--autoConnect`보다 `--browserUrl`이 exec 환경에서 더 안정적. 판매 가이드에 반영.

## #020 | 2026-03-30 | 에이전트가 "했다"고만 답하고 파일 미수정

## #023 | 2026-03-30 | 에이전트 기록 도구(폼) 정리

**발견:** 에이전트가 사용 가능한 파일 조작 도구 4가지 확인.
**도구 목록:**
| 도구 | 역할 | 사용 사례 |
|------|------|---------|
| `read` | 파일 읽기 | 재고현황 조회, CSV 확인 |
| `write` | 파일 생성/전체 덮어쓰기 | 새 보고서 생성, 빈 파일 초기화 |
| `edit` | 파일 부분 수정 | CSV 행 추가, 숫자 변경 (가장 자주 사용) |
| `apply_patch` | diff 패치 적용 | 복잡한 다중 위치 수정 |
**추가 도구:**
| `exec` | 셸 명령 실행 | 주문접수.sh 스크립트 호출, 외부 도구 실행 |
| `web_search` | 웹 검색 | 시장 조사, 경쟁사 정보 |
| `web_fetch` | 웹 페이지 가져오기 | 스마트스토어 크롤링 |
| `memory_search` | 기억 검색 | 과거 업무 기록 조회 |
| `message` | 메시지 전송 | 텔레그램 알림 전송 |
**핵심:** AGENTS.md에 도구 이름뿐 아니라 **어떤 업무에서 어떤 파일을 어떤 도구로 수정하는지** 구체적 절차를 적어야 에이전트가 실제로 실행함.
**일반화:** ✅ 판매 패키지 AGENTS.md 템플릿의 핵심 구성요소. 업무 매뉴얼 = AGENTS.md.
