## 🔧 도구 사용 규칙 (필수)

### 사용 가능한 도구
| 도구 | 용도 | 예시 |
|------|------|------|
| `read` | 파일 읽기 | `read("data/완제품재고.csv")` |
| `write` | 파일 생성/덮어쓰기 | `write("data/파일.csv", 내용)` |
| `edit` | 파일 부분 수정 | `edit("data/완제품재고.csv", 변경내용)` |
| `exec` | 셸 명령 실행 | `exec("bash ~/clawd/scripts/주문접수.sh ...")` |

### ⚠️ 핵심 규칙
1. **"했습니다"라고만 답하지 마라** — 반드시 도구를 호출하여 실제 파일을 수정하라
2. **수정 후 `read`로 검증하라** — 변경이 실제로 반영됐는지 확인
3. **CSV 형식 엄수** — 헤더 유지, 쉼표 구분, 기존 행 보존, 새 행 추가
4. **`재고현황.md`는 수정하지 마라** — 참고용 보고서일 뿐. 재고 데이터의 단일 원천(Single Source of Truth)은 `완제품재고.csv`이다. md 파일은 건드리지 말 것

### 업무별 파일 수정 방법

#### 출고 기록 시
1. `read("data/완제품재고.csv")` → 현재고 확인
2. 현재고 - 출고수량 = 새 재고 계산
3. `edit("data/완제품재고.csv")` → 해당 제품 현재고 수정
4. `edit("data/입출고이력.csv")` → 새 행 추가: `날짜,출고,코드,품명,수량,거래처,비고`
5. `read("data/완제품재고.csv")` → 변경 확인 후 사용자에게 보고

#### 입고 기록 시
1. `read("data/완제품재고.csv")` → 현재고 확인
2. 현재고 + 입고수량 = 새 재고 계산
3. `edit("data/완제품재고.csv")` → 해당 제품 현재고 수정
4. `edit("data/입출고이력.csv")` → 새 행 추가: `날짜,입고,코드,품명,수량,거래처,비고`
5. `read("data/완제품재고.csv")` → 변경 확인

#### 생산 기록 시
1. `read("data/완제품재고.csv")` → 현재고 확인
2. `edit("data/완제품재고.csv")` → 완제품 수량 증가
3. `edit("data/생산이력.csv")` → 새 행 추가
4. 부품 차감이 필요하면 `edit("data/부품마스터.csv")` → 부품 수량 감소

### 데이터 파일 형식

#### data/완제품재고.csv
```csv
제품코드,제품명,현재고,안전재고,단위
PG-001,퍼티스트 골프 고급형,82,30,대
```

#### data/입출고이력.csv
```csv
날짜,유형,코드,품명,수량,거래처,비고
2026-03-30,출고,PG-001,퍼티스트 골프 고급형,3,테스트골프장,주문ORD-20260330-003
```

#### data/생산이력.csv
```csv
날짜,제품코드,제품명,수량,작업자,비고
```

#### 제품코드 참조
| 코드 | 제품명 |
|------|-------|
| PG-001 | 퍼티스트 골프 고급형 |
| PG-002 | 퍼티스트 골프 일반형 |
| PP-001 | 퍼티스트 파크 고급형 |
| PP-002 | 퍼티스트 파크 일반형 |
| PJ-001 | 퍼티스트 일본형 |
| PM-001 | 퍼티스트 미국형 |

- **품질검사 체크리스트** - 품질관리 표준화
