# 셀러 온톨로지 — AI 온라인 회사의 두뇌

## 왜 필요한가

> "자동화에 집중하면 AI를 반밖에 못 쓴다. 컨텍스트가 쌓일수록 AI가 똑똑해지는 구조가 핵심이다."
> — 델타소사이어티 이웅재

개별 에이전트가 각자 일만 하면, 매번 처음부터 시작한다.
온톨로지가 있으면, **한 에이전트의 결과가 다른 에이전트의 입력**이 된다.

```
[컴파운드 효과]
소싱 에이전트가 상품 분석 → 상품 지식 축적
  → 콘텐츠 에이전트가 상세페이지 만들 때 상품 지식 참조 (더 정확한 카피)
  → 마케팅 에이전트가 SNS 만들 때 참조 (일관된 메시지)
  → CS 에이전트가 고객 질문에 답할 때 참조 (정확한 상품 정보)
  → 재무 에이전트가 마진 계산할 때 참조 (정확한 원가)
  → 풀필먼트가 재고 관리할 때 참조 (상품별 리드타임)

시간이 지날수록 → 지식이 쌓이고 → AI가 더 정확해지고 → 셀러의 승인률이 올라간다
```

## 온톨로지 파일 구조

```
workspace/ontology/
├── CORE.md                    ← 비즈니스 핵심 의사결정 (SST)
├── products/                  ← 상품 지식 베이스
│   ├── _index.md              ← 상품 목록 + 메타데이터
│   └── {product-id}.md        ← 상품별 상세 지식
├── customers/                 ← 고객 지식 베이스
│   ├── _patterns.md           ← 고객 행동 패턴
│   └── _faq.md                ← 자주 묻는 질문 + 최적 답변
├── channels/                  ← 채널별 규칙
│   ├── coupang.md             ← 쿠팡 수수료/정책/규격
│   ├── smartstore.md          ← 스마트스토어 규칙
│   └── 11st.md                ← 11번가 규칙
├── decisions/                 ← 의사결정 로그 (SST 개념)
│   └── YYYY-MM-DD.md          ← 일별 의사결정 기록
└── metrics/                   ← 성과 지표
    ├── daily.md               ← 일일 매출/마진/CS 통계
    └── weekly.md              ← 주간 종합 리포트
```

## 핵심 엔티티 정의

### 1. 상품 (Product)

```markdown
# 상품: {상품명}

## 기본 정보
- ID: {고유 ID}
- 카테고리: {의류/생활용품/전자기기/...}
- 소싱처: {타오바오/1688/국내도매/자체제작}
- 소싱가: {원}
- 판매가: {원}
- 마진율: {%}

## 소싱 분석 (소싱 에이전트가 축적)
- 경쟁 강도: {상/중/하}
- 시즌성: {연중/봄여름/가을겨울}
- 트렌드: {상승/안정/하락}
- 대안 소싱처: [{소싱처, 단가, 리드타임}]

## 콘텐츠 (콘텐츠 에이전트가 축적)
- 상세페이지 버전: {v1, v2, ...}
- 핵심 USP: {한 줄}
- 타겟 페르소나: {누구}
- SEO 키워드: [{키워드}]
- 전환율: {%} (추적 시)

## 마케팅 (마케팅 에이전트가 축적)
- 의인화 캐릭터: {성격, 말투}
- SNS 콘텐츠: [{플랫폼, URL, 반응}]
- 카드뉴스: [{제목, 성과}]
- 쇼츠: [{제목, 조회수}]

## CS 패턴 (CS 에이전트가 축적)
- 자주 묻는 질문: [{질문, 최적 답변}]
- 클레임 유형: [{유형, 빈도, 해결책}]
- 리뷰 키워드: {긍정: [...], 부정: [...]}

## 물류 (풀필먼트 에이전트가 축적)
- 현재 재고: {개}
- 일평균 판매: {개}
- 안전재고: {개}
- 리드타임: {일}
- 보관 위치: {3PL명/자체}
```

### 2. 고객 패턴 (Customer Pattern)

```markdown
# 고객 패턴

## 자주 묻는 질문 TOP 10
| 순위 | 질문 | 빈도 | 최적 답변 | 자동화 가능 |
|------|------|------|----------|-----------|
| 1 | 배송 언제? | 35% | {템플릿} | ✅ |
| 2 | 반품 어떻게? | 20% | {템플릿} | ✅ |
| ... | | | | |

## 구매 패턴
- 피크 시간대: {시}
- 재구매율: {%}
- 평균 장바구니: {원}
- 계절 패턴: {설명}

## 클레임 패턴
- 파손: {%} → 포장 개선 필요 여부
- 오배송: {%} → 출고 프로세스 점검
- 사이즈 불만: {%} → 상세페이지 사이즈 안내 보강
```

### 3. 비즈니스 의사결정 (SST — Single Source of Truth)

```markdown
# CORE.md — 비즈니스 핵심 의사결정

## 우리는 누구인가
- 사업: {한 줄 정의}
- 타겟: {핵심 고객}
- 차별점: {왜 우리를 써야 하는가}

## 현재 전략
- 주력 상품: [{상품}]
- 주력 채널: [{채널}]
- 가격 전략: {설명}
- 마케팅 전략: {설명}

## 의사결정 로그
| 날짜 | 결정 | 이유 | 결과 | 학습 |
|------|------|------|------|------|
| 2026-04-01 | A상품 소싱 결정 | 마진 36%, 경쟁 중간 | TBD | TBD |

## 금지 사항
- {하면 안 되는 것들 목록}
```

## 에이전트 간 데이터 흐름 규칙

### 축적 규칙 (Write)
각 에이전트는 작업 완료 시 반드시 온톨로지를 업데이트한다:

| 에이전트 | 축적 대상 | 트리거 |
|---------|----------|--------|
| 소싱 | `products/{id}.md` 소싱 분석 섹션 | 새 상품 분석 시 |
| 콘텐츠 | `products/{id}.md` 콘텐츠 섹션 | 상세페이지 생성 시 |
| 마케팅 | `products/{id}.md` 마케팅 섹션 | SNS/쇼츠 생성 시 |
| CS | `customers/_faq.md`, `products/{id}.md` CS 섹션 | CS 답변 승인 후 |
| 재무 | `metrics/daily.md` | 매일 정산 시 |
| 풀필먼트 | `products/{id}.md` 물류 섹션 | 재고 변동 시 |

### 참조 규칙 (Read)
각 에이전트는 작업 시작 전 반드시 온톨로지를 참조한다:

| 에이전트 | 참조 대상 | 이유 |
|---------|----------|------|
| 콘텐츠 | 상품 소싱 분석 | USP, 경쟁 포인트 반영 |
| 마케팅 | 상품 콘텐츠 + 고객 패턴 | 일관된 메시지, 타겟 맞춤 |
| CS | 상품 전체 + FAQ | 정확한 상품 정보로 답변 |
| 재무 | 상품 소싱가 + 채널 수수료 | 정확한 마진 계산 |
| 풀필먼트 | 상품 물류 + 판매 속도 | 정확한 품절 예측 |
| 소싱 | 재무 마진 + CS 클레임 | 수익성 + 품질 검증 |

### 컴파운드 파이프라인 (자동 체이닝)

```
[신상품 등록 파이프라인]
셀러: "이 상품 팔아줘" + URL

1. 소싱 에이전트
   → 마진 분석 + 경쟁 분석
   → products/{id}.md 생성 (소싱 섹션 채움)
   → 판정: 추천/주의/비추천

2. (추천인 경우) 콘텐츠 에이전트
   → products/{id}.md 참조
   → 상세페이지 자동 생성 (USP, 키워드 활용)
   → products/{id}.md 콘텐츠 섹션 업데이트

3. 마케팅 에이전트
   → products/{id}.md 참조 (소싱+콘텐츠)
   → 의인화 캐릭터 설정 + 쇼츠/카드뉴스 생성
   → products/{id}.md 마케팅 섹션 업데이트

4. 풀필먼트 에이전트
   → products/{id}.md 참조
   → 초기 재고 등록 + 안전재고 설정
   → products/{id}.md 물류 섹션 업데이트

5. 셀러에게 종합 보고:
   "상품 분석 → 상세페이지 → 마케팅 콘텐츠 → 재고 설정 완료.
    승인하시면 채널에 등록합니다."

[일상 운영 파이프라인]
매일 하트비트에서:
- CS: 문의 수집 → FAQ 업데이트 → 답변 초안
- 재무: 매출 수집 → 마진 계산 → daily.md 업데이트
- 풀필먼트: 재고 체크 → 품절 예측 → 발주 알림
- 소싱: 트렌드 모니터링 → 기회 발견 시 알림
```

## 온톨로지가 만드는 컴파운드 효과

```
1주차: 상품 3개, FAQ 5개, 기본 답변 템플릿
       → CS 정확도 60%, 셀러 수정률 40%

1개월차: 상품 15개, FAQ 50개, 셀러 수정 패턴 학습
         → CS 정확도 80%, 셀러 수정률 15%

3개월차: 상품 50개, FAQ 200개, 계절 패턴 축적
         → CS 정확도 92%, 셀러 수정률 5%
         → 소싱 추천 적중률 상승 (과거 마진 데이터 기반)
         → 상세페이지 전환율 상승 (리뷰 키워드 반영)

→ 시간이 지날수록 AI 조직이 "진짜 직원처럼" 셀러의 사업을 이해한다
→ 이것이 사방넷(매번 리셋) vs 우리(지식이 쌓임)의 궁극적 차별점
```
