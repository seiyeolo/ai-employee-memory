## 3. 1인/소규모 셀러의 풀필먼트 현황

### 사용 여부 판단 기준
- **월 출고 100건 이하:** 직접 포장/발송이 더 경제적
- **월 출고 100~500건:** 3PL 도입 고려 시점
- **월 출고 500건 이상:** 풀필먼트 필수 (직접 처리 불가능)

### 셀러의 풀필먼트 페인포인트
1. **어떤 업체를 골라야 하는지 모름** — 비교 정보 부족
2. **비용 구조가 불투명** — 건당 비용, 보관료, 추가 수수료 복잡
3. **입고/재고 관리 연동이 수동** — 엑셀로 재고 현황 파악
4. **반품/교환 처리 지연** — 3PL과 셀러 간 커뮤니케이션 병목
5. **다채널 재고 분산** — 쿠팡 물류, 네이버 풀필먼트 따로 관리

## 4. "AI 풀필먼트 에이전트" 기회

### 우리가 직접 물류를 하는 건 아님
> 풀필먼트 자체를 대행하는 게 아니라, **셀러와 풀필먼트 업체 사이를 AI가 연결하고 최적화하는 에이전트**

### 풀필먼트 에이전트의 역할

```
셀러: "이번 달 재고 어때?"
  → AI 풀필먼트 에이전트:
    ① 3PL 업체 API 실시간 재고 조회
    ② 채널별 판매 속도 분석 → 품절 예측
    ③ "3일 후 A상품 품절 예상. 발주 필요" 알림
    ④ 승인하면 → 자동 발주 + 입고 예약
    
셀러: "반품 들어왔어?"
  → AI 풀필먼트 에이전트:
    ① 3PL 반품 현황 조회
    ② 반품 사유 분류 + CS 답변 초안 생성
    ③ 재입고 가능 여부 판단
    ④ 승인하면 → 재입고 or 폐기 처리
```

### 연동 우선순위
1. **파스토** — 네이버 공식, 600개 채널 연동, API 가장 개방적
2. **품고** — AI 예측 모델 보유, 소셀러 친화적, 택배비 저렴
3. **위킵** — G마켓/옥션 스타배송, PrePack 자동화

### 기술 난이도
- **중간~높음** — 각 3PL의 API 규격이 다름, 표준화 작업 필요
- 파스토 FMS API가 가장 잘 문서화되어 있어 첫 연동 타겟으로 적합
